# 001 — build(flyway): V1 bet read-model 스키마

## 개요
정산이 참조할 bet 스냅샷을 담을 테이블을 만든다. **이 서비스는 betting-service DB를 직접 읽지 않는다**(ADR-0006 event sourcing) — `bet.placed.v1` 이벤트로 자체 read model을 구축할 것이므로, 그 저장소가 먼저 필요하다.

## 작업 순서
1. `bet` 테이블 — 스냅샷(slip 모양/stake) + 정산 lifecycle(상태/결과/payout)을 한 행에.
2. `bet_selection` 테이블 — 레그(event/market/selection/odds) + selection별 outcome.
3. 핫 스캔 인덱스 2개.

## 작업 내역
```sql
CREATE TABLE bet (
  bet_id PRIMARY KEY, user_id, slip_type,           -- SINGLE/MULTIPLE/SYSTEM
  system_min_wins, system_total_selections,         -- K, N (SYSTEM만)
  stake_amount, stake_currency,                      -- per-line(unit) stake
  status,                                            -- PENDING/SETTLED/VOIDED
  result, payout_amount, payout_currency,            -- terminal 시 채움
  requested_at, settled_at, created_at, updated_at);
CREATE TABLE bet_selection (
  selection_row_id PRIMARY KEY, bet_id REFERENCES bet,
  leg_index, event_id, market_id, selection_id,
  odds NUMERIC(9,4),                                 -- 제출가 (V1 payout 기준)
  outcome,                                           -- 정산 시 selectionId→결과 stamp
  UNIQUE(bet_id, leg_index));
CREATE INDEX ix_bet_selection_event ON bet_selection(event_id);   -- "이 event의 베팅들"
CREATE INDEX ix_bet_pending ON bet(status) WHERE status='PENDING'; -- partial index
```
- **stake는 per-line(unit) stake** — System은 C(N,K)개 line으로 펼쳐지므로 총 commit액 = unit × lineCount. betting과 같은 규약.
- **`ix_bet_pending`는 partial index** — `WHERE status='PENDING'`. PostgreSQL은 조건을 만족하는 행만 인덱싱하므로, 대부분 SETTLED가 된 큰 테이블에서도 "열린 베팅" 스캔이 작게 유지된다(C++엔 없는 DB 기능).

## 결과
스키마만. `ddl-auto=validate`라 다음 커밋의 엔티티가 이 스키마와 정확히 매칭돼야 컨텍스트가 뜬다.

## 요약
- `bet` + `bet_selection`, event/pending 인덱스.
- 스냅샷 + 정산 상태를 한 테이블에(STI-ish, JOIN 회피).

## 다음 작업
JPA 엔티티 + consumer로 이 스키마를 채운다.

## 핵심 확인
다음 커밋의 통합 테스트에서 Flyway가 이 마이그레이션을 적용(Testcontainers PostgreSQL).

## 기억/설명 Level
- **L1**: 두 테이블, 인덱스 2개.
- **L2**: partial index가 왜 적합한가, per-line stake 규약.
- **L3**: 왜 betting DB를 안 읽고 자체 read model인가 (ADR-0006 — 결합도/replay).
