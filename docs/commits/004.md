# 004 — feat(resolver): Single/Multiple/System 결과 판정 (ADR-0008)

## 개요
정산의 **도메인 핵심이자 면접 단골**. 베팅 슬립을 `SettlementResult`(WON/LOST/PUSH/VOID) + payout으로 판정한다. 의존성 순서상(순수 함수, 의존 0) 트리거·payout보다 **먼저** 짰다.

## 핵심 통찰 — 세 슬립 타입은 하나의 수학
Single/Multiple/System을 따로 짜지 않고 **line(누적) 모델**로 통일했다. betting의 `SystemBetCalculator`와 대칭이라, all-win 정산액이 접수 시점 max-payout과 정확히 일치한다.

```
payout = unitStake × Σ_lines Π_legs multiplier
```
- **line(누적)**: Single = 전체 1줄, Multiple = 전체 1줄(전부 살아야), System(K-of-N) = C(N,K)줄(각 K-부분집합).
- **multiplier(selection outcome별)**: WON = 그 odds, PUSH/VOID = **1.0**(누적에서 그 레그가 환불되어 빠짐 — ADR-0012 "void된 selection 빼고 재계산"), LOST = 그 line **사망**(곱 0).

## 작업 내역
```java
public SettlementOutcome resolve(BetSlipType type, List<ResolvedSelection> sels, Money stake) {
  BigDecimal summed = ZERO;
  for (List<Integer> line : lines(type, sels.size()))   // Single/Multi=[all], System=C(N,K)
    summed = summed.add(lineProduct(line, sels));        // LOST면 ZERO, 아니면 Π multiplier
  long payout = BigDecimal.valueOf(stake.amount()).multiply(summed)
                  .setScale(0, FLOOR).longValueExact();  // 단 한 번 floor (house 보수적)
  return new SettlementOutcome(classify(sels, payout), new Money(payout, stake.currency()));
}
```
- **classify**: payout 0 → LOST(모든 line이 죽음). payout>0 & WON 레그 존재 → WON. WON 없고 LOST 없음(전부 PUSH/VOID) → 전부 VOID면 VOID, 아니면 PUSH(둘 다 stake 환불). 이 한 함수가 Single까지 커버한다(Single = 1 line).
- **rounding**: `setScale(0, FLOOR)`로 합산된 곱에 **단 한 번** floor. betting의 max-payout과 같은 규약이라 정합. C++ 관점: `BigDecimal`은 임의정밀 십진(odds 1.85 같은 값의 부동소수 오차 회피) — `double`로 돈 계산하면 안 되는 것과 같은 이유.
- **half-won/half-lost 미구현**: ADR-0012 V1 제외. selection은 항상 통짜 4결과이므로 fractional multiplier가 없다(주석으로 명시).

## 요약
- 세 타입을 line 모델로 통일 (betting과 대칭).
- multiplier: WON=odds, PUSH/VOID=1.0, LOST=line 사망.
- floor 한 번, classify로 슬립 결과 분류.
