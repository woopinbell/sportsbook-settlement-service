# 017 — test(api): web-layer 커버리지

## 개요
`@WebMvcTest`로 컨트롤러 + RFC 7807 응답을 빠르게(DB/Kafka 없이) 검증.

## 작업 내역 (8 케이스)
- `@WebMvcTest(SettlementController.class)` + `@MockBean`(BetRepository/SettlementService/SettlementTrigger). 풀 컨텍스트 없이 web 슬라이스만 → 빠름.
- GET: settled 베팅 뷰(status/result/payout) / 미존재 404 `application/problem+json`.
- replay: 202(결과 있음) / 404(없음).
- void: 200(PENDING) / 409(비-PENDING) / 404(미존재).
- malformed UUID → 400 VALIDATION_FAILED.
- **버그 잡음**: 처음 `$.code`로 단언했으나 shared-protocol `ProblemDetail`은 JSON에 **`errorCode`**로 직렬화 → `$.errorCode`로 수정. (4건 실패 → 통과)

```java
@WebMvcTest(SettlementController.class)
class SettlementControllerTest {
  @MockBean BetRepository bets; @MockBean SettlementService settlement; @MockBean SettlementTrigger trigger;
  ...
  mvc.perform(get("/internal/v1/settlements/{betId}", betId))
     .andExpect(status().isNotFound())
     .andExpect(jsonPath("$.errorCode").value("NOT_FOUND"));
}
```

## 결과
8 테스트 green. 상태/replay/void 경로 + 문제 응답 shape 검증.

## 요약
- `@WebMvcTest` + MockBean, 8 케이스.
- ProblemDetail JSON 필드는 `errorCode`.
