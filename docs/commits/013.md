# 013 — build(flyway): V4 materialized match-result 스키마

## 개요
MatchResult가 전달한 selection별 결과를 영속화한다. 두 가지를 가능하게: (a) **replay**(admin이 이벤트 재정산), (b) **late-arriving bet**(BetPlacedRequested가 결과보다 늦게 와도 stored 결과로 정산).

## 작업 내역
```sql
CREATE TABLE match_result (
  event_id PRIMARY KEY, mode,                        -- COMPLETED/ABANDONED/VOIDED
  settled_at, received_at);
CREATE TABLE match_selection_result (
  event_id REFERENCES match_result, selection_id, outcome,   -- 계약 결과
  PRIMARY KEY (event_id, selection_id));
```
- `match_result` + 자식 `match_selection_result` — JPA `@ElementCollection Map<UUID,SettlementResult>`로 매핑(엔티티에서 selectionId→결과 맵).
- 24h 윈도우 내 정정(ADR-0012)은 event_id upsert.
- C++ 관점: `@ElementCollection`은 엔티티가 아닌 값들의 컬렉션을 별도 조인 테이블에 매핑(소유 엔티티 생명주기에 종속) — ORM 특유의 개념.
