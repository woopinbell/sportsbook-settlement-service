# 015 — feat(dlq): retry backoff → dead-letter

## 개요
정산이 계속 실패하면(예: wallet 다운) 무한 재시도로 막히지 않게, 3회 backoff 후 `<topic>.DLT`로 격리하고 운영 메트릭을 올린다(ADR-0006).

## 작업 내역
- **`DefaultErrorHandler` 빈**(Spring Boot가 listener container factory에 자동 적용): `FixedBackOff(backoffMs, maxAttempts-1)`로 N회 재시도 후 `DeadLetterPublishingRecoverer`가 `<topic>.DLT`(같은 key/partition)로 republish. recoverer를 감싸 `settlement.dlq` 카운터 증가(운영 알람).
- **poison pill 즉시 격리**: `addNotRetryableExceptions(IllegalArgumentException.class)` — bad UUID/unknown result 같은 파싱 오류는 재시도해도 소용없으니 바로 DLT.
- **부분 실패 안전**: 정산이 DLT로 가도 베팅은 **half-settled가 아니라 PENDING**으로 남는다(2-phase finalize 전에 실패하면 전이 안 됨) → admin replay 대상.

```java
DefaultErrorHandler h = new DefaultErrorHandler(
    (record, ex) -> { meter.counter("settlement.dlq","topic",record.topic()).increment();
                      dlq.accept(record, ex); },
    new FixedBackOff(backoffMs, maxAttempts - 1));
h.addNotRetryableExceptions(IllegalArgumentException.class);
```

## 결과
실패 정산이 DLT로 격리됨을 통합 테스트로 증명(wallet 503 → `match.result.DLT`에 도착, 베팅 PENDING 유지).

## 요약
- 3회 backoff → `<topic>.DLT` + `settlement.dlq` 메트릭.
- 파싱 오류는 non-retryable(즉시 DLT).
- 실패 베팅은 PENDING(half-settled 아님).
