# 000 — chore(project): settlement-service scaffold

## 개요
정산 서비스의 빈 골격을 세운다. Phase 4 첫 repo이므로 betting/wallet의 검증된 레이아웃을 그대로 미러링하되, **이 서비스에 불필요한 것은 뺀다**. 핵심 결정: shared-protocol `0.2.0-SNAPSHOT`(BetSettled/BetVoided 추가본) 의존, **Redis 미포함**(정산 멱등은 캐시가 아니라 DB unique 제약이 보증), 포트 8084.

## 작업 순서
1. `git init` (이 repo는 독립 git repo).
2. `.gitignore` — Java/Maven 표준 + **`CLAUDE.md` 라인 필수**.
3. `pom.xml` — Boot 3.2.11 parent, Java 17, shared-protocol 0.2.0, spring-kafka/avro/data-jpa/flyway/web/validation/aop/resilience4j/observability/lombok + test(JUnit5/AssertJ/Mockito/Testcontainers/WireMock).
4. `config/checkstyle/checkstyle.xml`, `SettlementServiceApplication`, `application.yml`/`application-test.yml`/`logback-spring.xml`.
5. `docs/commits`·`docs/reflection`에 `.gitkeep`만 (docs/notes 미생성 — 2026-05-29 정책).
6. `mvn verify` green 확인 후 단독 커밋.

## 작업 내역
- **pom 차별점**: betting과 거의 동일하나 `spring-boot-starter-data-redis`를 뺐다. 정산 멱등은 DB `bet_id` PK + 상태 가드로 강하게 보증하므로 캐시 가속이 불필요하고, 안 쓰는 Redis 의존은 런타임에 phantom 연결만 만든다.
- **application.yml**: Kafka consumer `manual_immediate` ack + `ByteArrayDeserializer`(Avro raw bytes), producer idempotence, `settlement.topics.*`(bet-placed=`bet.placed.v1`, match-result, event-lifecycle, bet-settled, bet-voided), wallet client, outbox poll, retry/DLQ, late-settlement 24h, resilience4j walletClient breaker.
- **C++ 관점**: `pom.xml`은 CMake보다 선언적이다. `spring-boot-starter-parent`가 **BOM**(Bill of Materials)으로 수십 개 라이브러리 버전을 한꺼번에 고정한다 — 직접 버전을 안 적어도 호환 조합이 보장된다(C++의 vcpkg/conan lockfile과 비슷하나 transitive 충돌 해결까지 포함).
