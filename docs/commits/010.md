# 010 — test(settlement): WireMock 기반 payout/void 증명

## 개요
정산 서비스의 wallet 연동을 실제 PostgreSQL + WireMock wallet으로 증명. 돈이 걸린 경로이므로 각 결과 타입의 credit leg를 정확히 검증.

## 작업 내역 (6 케이스)
- **WON single**: SETTLED·WON·payout 20000. WireMock에 **2건** credit — `settle:refund:<betId>`(USER_LOCKED, stake) + `settle:payout:<betId>`(HOUSE_POOL, 이익). BetSettled outbox(result=WON, payout=20000, stake=10000) 검증. **재전달 → 정확히 2 credit + outbox 1행**(멱등).
- **LOST single**: SETTLED·LOST·payout 0, **credit 0건**(이미 차감), BetSettled(LOST).
- **settle VOID**: selection VOID → SETTLED·VOID, `settle:refund` 1건(USER_LOCKED, stake 환불).
- **voidOnEvent**: VOIDED, `void:refund:<betId>` 1건, **BetVoided**(reason EVENT_CANCELLED, refund) outbox.
- **멀티이벤트 readiness**: A 결과만 → PENDING 유지·credit 0·outbox 0. B 결과 → 정산 WON payout 30000(10000×2×1.5).

```java
wm.verify(postRequestedFor(CREDIT)
   .withHeader("Idempotency-Key", equalTo("settle:refund:"+betId))
   .withRequestBody(matchingJsonPath("$.source", equalTo("USER_LOCKED"))));
// 재전달 후:
wm.verify(exactly(2), postRequestedFor(CREDIT));   // 2N 아님
assertThat(outbox.count()).isEqualTo(1L);
```
- **WireMock 한계 주의**: 스텁은 키로 dedup하지 않으므로(실제 wallet은 함) 동시성 단언은 "정확히 2건"이 아니라 "정산 1회 = outbox 1행 + 고정 키 사용"으로 한다(중복 payout 방지는 wallet 멱등이 보증).

## 결과
6 테스트 green. WON 2-leg, LOST 무호출, VOID 환불, 멀티이벤트, 멱등 재정산 모두 증명.

## 요약
- 각 결과의 credit leg를 키·source까지 검증.
- 재전달 멱등(2 credit, outbox 1).
