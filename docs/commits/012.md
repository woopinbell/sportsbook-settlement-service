# 012 — test(trigger): match-result settles / cancelled voids / finished no-op

## 개요
토픽에 실제 Avro 이벤트를 publish해 read model 상태 전이를 end-to-end 검증.

## 작업 내역 (3 케이스, @SpringBootTest + Testcontainers PG + EmbeddedKafka + WireMock)
- **MatchResult(COMPLETED, {sel:WON})** → `match.result` publish → 베팅 SETTLED·payout 20000 await.
- **EventLifecycle(CANCELLED)** → `event.lifecycle` publish → 베팅 VOIDED await.
- **EventLifecycle(FINISHED)** → 베팅 PENDING 유지(윈도우 동안) — 정보성 확인.

```java
MatchResult result = new MatchResult(eventId.toString(), "2-1",
    MatchFinalStatus.COMPLETED, Map.of(selectionId.toString(), "WON"), settledAt);
send("match.result", eventId.toString(), result);
await().atMost(15s).untilAsserted(() ->
    assertThat(bets.findById(betId).orElseThrow().status()).isEqualTo(SETTLED));
```

## 결과
3 테스트 green. consumer→trigger→service→read model 전이가 실제 브로커로 증명.

## 요약
- MatchResult→SETTLED, CANCELLED→VOIDED, FINISHED→no-op.
- 실제 Avro 직렬화 + Kafka + DB 왕복.
