# 005 — test(resolver): void 재계산 + System 부분 적중 증명

## 개요
판정 로직은 면접 단골이자 돈이 걸린 코드 — 테스트로 정확성을 못 박는다. 21개 케이스로 세 슬립 타입을 전부 커버.

## 작업 내역 (@Nested로 그룹)
- **Single (5)**: WON(stake×odds), LOST(0), PUSH(환불), VOID(환불), **floor**(1001×1.85=1851.85 → 1851).
- **Multiple (7)**: 전부 WON(곱), 하나라도 LOST→LOST(0), **VOID 레그 탈락 후 생존 레그로 재계산**(`[WON 2.0, VOID]`→20000), PUSH 레그 동일, 전부 VOID→VOID(환불), WON 없는 PUSH+VOID→PUSH(환불), 3레그 중 1 VOID→나머지 2로 재계산.
- **System 2-of-3 / 1-of-4 (7)**: 전부 WON(3 line 합=26000), **부분 적중**(sel2 LOST → line{0,1}만 생존 6000), 1승만(2-of-3 충족 못 함→LOST), VOID 레그(line이 1.0으로 통과 14000), 전부 LOST(0), 전부 VOID(unit×C(3,2) 환불 3000), 1-of-4(승리 fold만 지급).
- **combinations (2)**: `C(3,2)=[[0,1],[0,2],[1,2]]`, `C(4,1)` 검증.

핵심 케이스 예:
```java
// Multiple: VOID 레그가 빠지고 생존 레그로 재계산 (ADR-0012)
resolve(MULTIPLE, [sel("2.0",WON), sel("1.5",VOID)], krw(10000))
  → result WON, payout 20000   // 10000 × 2.0 × 1.0

// System 2-of-3 부분 적중: 살아남은 line만 지급
resolve(System(2,3), [WON 2.0, WON 3.0, LOST 4.0], krw(1000))
  → result WON, payout 6000    // line{0,1}=2×3 생존, {0,2}{1,2} 사망
```
